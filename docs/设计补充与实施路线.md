# CMP 设计补充与实施路线

## 1. 当前定位

LocalCMP 阶段一采用模块化单体：一个 Node.js API 服务承载账户、用户权限、SIM/eSIM、套餐、CDR、Billing、批量任务、审批、审计和 Dashboard 模块。数据库设计使用多 schema PostgreSQL，业务代码保持清晰模块边界，后续可以按 Supplier Integration、CDR Ingestion、Rating、Billing、eSIM Orchestration 等领域拆分微服务。

## 2. 阶段一已落地能力

- 账户树与 Reseller 分级：支持 platform、reseller、customer，包含账户业务状态、风控状态、账务状态，以及父级限制继承。
- 用户、角色、权限：支持用户列表、角色权限查询、用户停用，新增请求上下文和权限校验模拟。
- SIM/号码生命周期：支持 activate、suspend、resume、terminate 的预览与执行，包含库存状态、账户状态、套餐授权、并发操作校验。
- 套餐与订阅：支持套餐状态流转、套餐授权、订阅、流量池展示，退市影响存量订阅时进入审批。
- CDR 标准化：支持供应商 CDR 记录导入、ICCID 校验、重复 supplier_record_id 去重、统一输出标准 CDR。
- Billing：支持账单查询、账单运行预览、账单邮件发送入队模拟。
- 批量任务：支持创建批量任务、生成行级任务、执行并记录结果，事件进入 outbox。
- 客户集成：支持 API Client 创建、密钥轮换、暂停，Webhook 订阅、测试投递和通知投递查询。
- 审批、审计与 outbox：高风险动作可生成审批单，关键写操作写入 audit logs 和 outbox events。
- 前端联动：静态 CMP 管理台可调用后端 API 展示 Dashboard、账户、用户、套餐、SIM、CDR、账单、审批等数据。

## 3. 横切架构补充

开发期使用 Header 模拟认证：

- `X-User-Id`：默认 `user_scott`。
- `X-Account-Id`：默认 `acc_root`。
- `X-Correlation-Id`：用于 API、审计、outbox 和供应商调用链追踪。
- `Idempotency-Key`：用于 SIM/eSIM、账单、批量等写操作的幂等控制。

生产环境应替换为 OIDC/OAuth2、API Client Credentials、mTLS 和 API Gateway 注入的可信上下文。

权限采用 `action.resource` 风格，例如 `sim.read`、`sim.operate`、`package.manage`、`usage.import`、`billing.manage`、`approval.decide`、`audit.read`。阶段一在 API 路由层做粗粒度校验；生产环境需要补充账户树数据权限、字段权限、审批权限、导出权限和供应商成本字段脱敏。

## 4. 状态机原则

- Controller 不直接改最终业务状态，只调用领域服务。
- 状态流转前必须做业务前置校验。
- 写操作必须记录 audit log。
- 异步副作用通过 outbox event 表示。
- 同一个 SIM 同一时间只允许一个未完成服务操作。

## 5. 近期实现优先级

1. 接入真实 PostgreSQL：增加 repository 层、迁移脚本、事务、连接池、RLS 策略和种子数据。
2. 完整认证鉴权：登录、Session/JWT、API Client、Scope、MFA、SSO、Support 临时授权。
3. 完整 eSIM 编排：eID/eUICC、Profile 状态机、SM-DP+/eIM Adapter、回调与补偿查询。
4. CDR Pipeline：原始文件、Parser 版本、行级错误、Rating、流量池扣减、出账锁定。
5. Billing 自动化：账期生成、发票 PDF、邮件模板、发送重试、对账和调整单。
6. 批量任务生产化：文件上传、模板校验、审批流、并发控制、失败重试、结果下载。
7. 客户开放 API 产品化：正式 OAuth2 Client Credentials、OpenAPI 发布、SDK 示例、Webhook 签名校验示例、限流和客户沙箱。
8. 运维能力：健康检查、指标、日志、告警、备份恢复、灰度发布和 SLA 报表。

## 6. MVP 必要补充边界

为避免阶段一过重，后续只优先推进会影响可用闭环的能力：

- 真实 PostgreSQL 持久化：当前已提供 `DATABASE_URL` 驱动的 PostgreSQL JSONB 持久层，下一步再拆正式 repository 和领域表。
- 基础登录与 Token：当前已支持邮箱密码登录和 Bearer JWT；Header 模拟登录默认关闭，仅 `ALLOW_HEADER_AUTH=true` 时可用于本地调试。
- 前端运行时配置：通过 `frontend/config.js` 支持云端 API 地址配置。
- SIM Operation 查询闭环：操作提交后可查询状态、失败原因和审计链路。
- eSIM Profile 基础操作：先覆盖 allocate、enable、disable、release，不引入真实 SM-DP+/eIM。
- CDR、Billing、Batch 前端最小闭环：先支持页面触发、结果反馈、刷新列表。
- 本地冒烟脚本：让前端和后端人员能用一个命令验证核心链路。

非必要暂缓：复杂 Rating、完整税务发票、SAML/SCIM、字段级权限设计器、多区域容灾、供应商真实生态适配和高级 BI 报表。
