# 物联网 CMP 连接管理平台中文总体设计

## 1. 平台定位

本平台定位为云原生物联网流量连接管理平台，即 CMP。目标客户包括运营商、MVNO、物联网解决方案商、流量资源转售商、企业客户和多级 Reseller。平台提供从 SIM/eSIM 生命周期管理、资源方流量对接、套餐售卖、CDR 话单统一化、计费账单、客户 API 到运营 Dashboard 的完整能力。

平台设计参考国际主流 CMP 产品能力，例如 Jasper/Cisco IoT Control Center 类产品强调的连接可视化、Provisioning、移动服务管理、诊断、Billing、业务自动化、运营商集成和客户自助能力。同时结合 eSIM 的 GSMA RSP 标准体系，支持面向消费设备、M2M 和 IoT 场景的 profile 管理和远程操作。

## 2. 设计目标

- 专业：功能覆盖标准 CMP 的核心场景，支持 SIM、eSIM、码号、套餐、计费、资源方、客户 API 和运营后台。
- 强大：支持多资源方、多级账户、多套餐策略、流量池、批量操作、自动化账单和大规模 CDR 处理。
- 云端适配：采用云原生架构，支持弹性扩容、多可用区部署、异步任务、消息队列、缓存、对象存储和观测体系。
- 高性能：常用查询走聚合表和缓存，大规模 CDR 走分区表/数据仓库，业务操作异步化。
- 高可用：支持重试、幂等、死信队列、供应商熔断、降级、备份恢复、SLA 监控。
- 低耦合：按领域服务拆分，模块升级时减少交叉影响。
- 可集成：给客户提供标准 OpenAPI、Webhook、批量导入导出和沙箱环境。

## 3. 角色与权限模型

平台建议采用账户层级加 RBAC 权限的组合模型。对大型客户和多级代理场景，可进一步增加 ABAC 属性权限，例如地区、SIM 组、套餐组、成本中心、账户树范围。

主要角色：

- 平台超级管理员：管理全局配置、供应商、资源、套餐模板、账单策略、SLA、系统设置。
- 平台运营：处理 SIM/eSIM 操作、批量任务、资源导入、CDR 异常、供应商工单。
- 平台财务：管理价格、账单、发票、结算、调整单、信用额度。
- Reseller 管理员：创建和管理下级账户，分配套餐，查看下级用量和账单。
- 企业客户管理员：管理本账户下 SIM/eSIM、套餐、用户、API Key、告警和账单。
- 开发者：管理 API Key、Webhook、接口调用日志和集成配置。
- 只读审计员：查看数据、日志和账单，不允许做变更操作。

## 4. 账户体系

账户采用树形结构：

- 平台根账户
- 一级 Reseller
- 多级 Reseller
- 企业客户
- 子账户、部门、项目、成本中心

账户核心能力：

- 支持父子账户关系和无限级或配置级数的下级账户。
- 支持 Reseller 给下级账户分配可售套餐、价格、额度和 API 权限。
- 支持下级用量向上级汇总，用于 Reseller 结算和利润分析。
- 支持账户级币种、时区、账期、信用额度、税务信息、合同周期和发票邮箱。
- 支持账户数据隔离，用户只能访问授权账户树范围内的数据。

账户状态建议拆成业务状态、风控状态和财务状态。业务状态包括 draft、pending_review、active、suspended、closed；风控状态包括 normal、credit_hold、compliance_hold、fraud_hold；财务状态包括 current、overdue、dunning、bad_debt。具体账户创建、审批、暂停、恢复、关闭、Reseller 层级授权和状态切换规则见 [账户用户权限套餐与号码状态深化设计.md](账户用户权限套餐与号码状态深化设计.md)。

## 5. eSIM 管理

eSIM 模块是平台的高价值核心模块。建议将 eUICC/eID、Profile、Profile Operation、供应商 RSP 能力拆开建模。

核心对象：

- eID/eUICC：设备或芯片唯一标识，记录所属账户、设备、状态、供应商和能力。
- eSIM Profile：可下载或已安装的运营商 profile，关联 ICCID、IMSI、MSISDN、套餐和资源方。
- Profile 操作：下载、安装、启用、禁用、删除、切换、释放、状态刷新。
- RSP 配置：SM-DP+、SM-SR、eIM、Activation Code、Matching ID、证书和供应商通道。

能力要求：

- Profile 库存导入、分配、回收和状态同步。
- 按 eID 绑定 profile，支持一机多 profile 场景。
- 支持批量 profile 操作，并带有预校验、异步执行、结果下载和失败重试。
- 支持 SGP.22、SGP.02、SGP.32/IoT eIM 方向的扩展适配。
- 所有 eSIM 操作必须有状态机、幂等键、供应商交易号、审计日志和回调事件。

## 6. SIM 与码号管理

SIM/码号模块负责传统实体 SIM、eSIM profile 对应号段和网络身份的生命周期。

管理字段：

- ICCID、IMSI、MSISDN、eID、IMEI、APN、静态 IP、运营商、MCC、MNC、国家、供应商。
- SIM 状态：库存、已分配、待开通、测试、已激活、暂停、停机、销户、回收、报失、废弃。
- 网络策略：漫游范围、接入制式、APN 策略、限速策略、IMEI 绑定策略。

核心能力：

- 码号导入、分配、回收、冻结、转移和生命周期跟踪。
- SIM 详情页展示套餐、用量、操作记录、网络诊断、CDR 和账单影响。
- 号段管理，支持按供应商、国家、运营商、批次、状态、账户筛选。
- 支持异常检测，例如长时间无流量、突增流量、异常漫游、IMEI 变化。

## 7. 资源方对接与流量转售

资源方包括 MNO、MVNO、漫游聚合商、eSIM 聚合商、批发流量商、SM-DP+/eIM 服务商和 CDR 提供方。

资源方集成能力：

- 供应商适配器框架，每个供应商一个 Adapter，支持版本化和独立配置。
- 供应商产品同步，包括国家、运营商、网络制式、APN、区域、价格、有效期、限速和公平使用策略。
- 开通接口：激活、暂停、恢复、销户、套餐变更、状态查询。
- eSIM 接口：profile 分配、下载、启用、禁用、删除、状态同步。
- CDR 获取：SFTP、API Pull、Webhook、对象存储投递、消息队列。
- 结算对账：供应商成本与平台内部计费结果进行差异分析。

CDR 统一化原则：

- 保留原始文件和原始记录 hash，保证可追溯。
- 将不同供应商格式转成平台统一 CDR。
- 同一条供应商记录必须可去重。
- 统一 CDR 进入 Rating 引擎，形成客户可下载的标准话单。

## 8. 套餐与计费策略

套餐模块需要支持灵活配置，避免每个客户都通过定制代码实现。

套餐维度：

- 地区：国家、区域、运营商、MCC/MNC、自定义 Zone、全球。
- 流量：固定流量、无限流量带 FUP、按量计费、阶梯价格、共享池。
- 周期：日包、月包、年包、一次性包、自定义周期、自动续订。
- 起算方式：自然月、开通当日、首次使用当日、合同指定日。
- 共享方式：单 SIM、账户池、Reseller 池、设备组池、跨区域池。
- 超额策略：断网、限速、按量收费、自动加购、仅告警。

Rating 规则：

- Rate Plan 必须版本化，历史账单按当时版本计费。
- 优先级建议为：客户合同价 > Reseller 分配价 > 平台默认销售价 > 供应商成本价。
- 支持预付费余额、后付费账单、信用额度和押金。
- 支持用量准实时计费和月底批量结算。

套餐对象建议分为 Package Template、Rate Plan、Package Entitlement、Subscription 和 Usage Pool。套餐状态包括 draft、pending_review、active、deprecated、retired、archived；订阅状态包括 pending_activation、active、suspended、pending_change、expired、cancelled、terminated。套餐发布、价格版本、订阅开通、变更、续期、下架和流量池扣减规则见 [账户用户权限套餐与号码状态深化设计.md](账户用户权限套餐与号码状态深化设计.md)。

## 9. Billing 账单能力

Billing 模块需要覆盖客户账单、Reseller 结算和供应商成本结算。

能力范围：

- 自动创建账户时同步创建 Billing Profile。
- 按周期生成账单，支持自然月、开通日、合同日和自定义周期。
- 账单预览、审批、发布、邮件发送、下载和作废/调整。
- 支持发票 PDF、账单明细 CSV、统一格式 CDR 文件。
- 支持折扣、税费、最低消费、封顶、超额费用、一次性费用、调整项。
- 支持 Reseller 下级客户账单汇总和利润分析。
- 支持供应商账单与内部 CDR 成本对账。

## 10. Dashboard 与报表

Dashboard 分为平台视角、Reseller 视角和客户视角。

平台视角：

- 活跃 SIM、eSIM profile、账户数、资源方状态、当月用量、收入、成本、毛利。
- CDR 导入延迟、Rating 积压、账单任务状态、失败操作、供应商 API 成功率。

客户视角：

- 当前套餐、剩余流量、流量池使用率、Top SIM、区域用量、告警、账单状态。
- eSIM 操作统计，例如成功、失败、处理中、供应商异常。

运营视角：

- 批量任务进度、失败原因分布、异常 CDR、供应商调用延迟、队列积压。

## 11. 批量操作

批量能力适用于高频运营场景。

支持操作：

- 批量 SIM/eSIM 导入。
- 批量开通、暂停、恢复、销户。
- 批量绑定套餐、变更套餐、关闭套餐。
- 批量 eSIM profile 下载、启用、禁用、删除。
- 批量资源导入、CDR 重跑、价格迁移、账户创建。

执行流程：

1. 上传文件。
2. 结构校验。
3. 业务校验。
4. 预览影响。
5. 审批确认。
6. 异步执行。
7. 查看进度。
8. 下载结果。

每一行记录都需要结果状态、错误码、错误原因、重试标记和操作关联 ID。

## 12. Settings 设置中心

Settings 用于统一管理平台级和租户级配置。

配置类型：

- 供应商鉴权：API Key、Token、OAuth、证书、SFTP Key、IP 白名单。
- 客户集成：API Key、OAuth Client、Webhook、签名密钥、限流策略。
- 系统参数：币种、税率、时区、账期、邮箱模板、短信模板、对象存储、SLA 阈值。
- 安全策略：MFA、密码策略、登录限制、IP 白名单、会话时长。
- 通知策略：账单邮件、告警邮件、Webhook 事件、运营工单通知。

敏感配置必须加密存储，并支持轮换、禁用和审计。

## 13. 客户 API 能力

客户 API 是 CMP 平台商业化和自动化的重要能力。

接口范围：

- 账户、用户、角色、API Key。
- SIM/eSIM 查询与操作。
- 套餐开通、变更、关闭。
- 用量查询、CDR 导出、账单查询。
- 批量任务创建和查询。
- Webhook 订阅管理。

设计要求：

- 使用 OAuth2 Client Credentials 或签名 API Key。
- 所有写操作支持 Idempotency-Key。
- 所有响应带 request_id/correlation_id。
- 支持按账户限流。
- 提供 OpenAPI 文档和沙箱环境。
- Webhook 支持签名、重放保护和失败重试。

## 14. 云原生架构

推荐采用模块化微服务或模块化单体加服务化演进的架构。初期如果团队规模有限，可以先采用模块化单体，但必须保持领域边界、数据库边界和异步消息边界清晰，后续可平滑拆分。

核心组件：

- API Gateway：认证、限流、路由、WAF、日志。
- IAM 服务：用户、角色、权限、SSO、API Key。
- Account 服务：账户树、Reseller、合同、额度。
- SIM Inventory 服务：SIM、码号、设备、网络策略。
- eSIM Orchestration 服务：profile、eID、RSP 操作、状态机。
- Supplier Integration 服务：供应商 Adapter、产品同步、调用日志。
- CDR Ingestion 服务：原始文件、解析、标准化、去重。
- Rating 服务：套餐计费、用量计费、流量池。
- Billing 服务：账单、发票、邮件、调整项、结算。
- Batch Job 服务：导入、校验、异步执行、结果。
- Notification 服务：邮件、Webhook、告警。
- Analytics 服务：Dashboard、聚合报表。
- Audit 服务：审计、合规、操作日志。

## 15. 可扩展与 SLA

高并发和大量用户场景下，需要将长耗时和高吞吐任务异步化。

关键设计：

- CDR、Rating、Billing、批量操作全部走消息队列和后台任务。
- Dashboard 查询走聚合表，不直接扫原始 CDR。
- 常用数据使用 Redis 缓存。
- 大表按月份、供应商、账户进行分区。
- 对供应商接口加熔断、限流、重试和降级。
- 对每个外部调用记录 latency、status、payload hash 和 correlation_id。
- 支持多副本、多可用区、自动扩容、健康检查和滚动升级。

建议 SLA 目标：

- 门户可用性：99.9% 起步，成熟后 99.95%。
- API 可用性：99.9% 起步。
- CDR 入库延迟：常规场景 5-30 分钟内。
- eSIM/SIM 操作提交：秒级返回受理结果，最终状态异步更新。

## 16. 模块解耦原则

- 服务之间通过 API、事件和领域对象交互，不共享内部实现。
- 供应商差异集中在 Adapter 层，核心业务只依赖标准接口。
- eSIM 操作通过状态机驱动，不让页面或供应商回调直接改终态。
- Rating 与 Billing 解耦，Rating 产出已计费明细，Billing 消费明细生成账单。
- CDR 原始数据、标准化数据、计费数据、账单数据分层保存。
- 前端按业务域拆模块，避免所有页面依赖同一个巨型状态。

## 17. 建议阶段规划

第一阶段：

- 账户树、用户权限、SIM 库存、套餐配置、资源方 Adapter 框架、CDR 标准化、基础 Dashboard、基础 Billing、客户 API、审计。

第二阶段：

- 完整 eSIM 编排、批量操作、流量池、高级 Rating、账单邮件自动化、供应商对账、Webhook、API 沙箱。

第三阶段：

- 多资源方智能路由、实时诊断、异常检测、SLA 自动化、独立租户/分片、数据仓库、Reseller Marketplace。
