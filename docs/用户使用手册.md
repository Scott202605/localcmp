# CMP 用户使用手册

## 1. 登录与权限

用户登录后进入 CMP 控制台。阶段一原型已支持基础邮箱密码登录和 Bearer JWT。

开发默认账号：

```text
邮箱：13911657767@139.com
密码：LocalCMP@2026
```

该账号仅用于本地和演示环境，生产部署前必须修改密码、配置 `JWT_SECRET`，并关闭 Header 模拟登录。

不同角色看到的菜单和按钮不同：

- 平台管理员：账户、用户、套餐、供应商、账单、审计、设置全部可见。
- Reseller 管理员：可管理自己及下级客户，不能查看平台成本和其他 Reseller 数据。
- 客户操作员：可查看并操作本账户 SIM/eSIM。
- 客户财务：可查看用量、账单和导出明细。
- 只读用户：仅能查看被授权范围内的数据。

## 2. Dashboard

Dashboard 展示 SIM 总量、活跃数量、暂停数量、月度流量、eSIM 操作成功率、CDR 延迟、供应商健康状态和账单待处理金额。

## 3. 账户管理

账户分为 platform、reseller、customer 和 sub-account。常见流程：

1. 创建账户基础资料。
2. 配置账期、币种、信用额度和发票邮箱。
3. 分配可售套餐和价格底线。
4. 配置安全策略和 API 权限。
5. 提交审核。
6. 审核通过后账户进入 active。

账户状态：draft、pending_review、active、suspended、closed。关闭账户前必须先处理活跃 SIM、未完成账单、批量任务和下级账户。

## 4. 用户与权限

管理员可以邀请用户、分配角色、停用用户。用户离职或外部接口停用时，应立即停用账号并撤销关联 API Client。

高风险权限建议单独授予：批量终止 SIM、发布或退市套餐、发送正式账单、修改供应商鉴权配置、导出全量 CDR。

## 5. SIM/号码管理

SIM 列表展示 ICCID、IMSI、MSISDN、eID、账户、供应商、库存状态、服务状态、套餐和本月流量。

服务状态：

- not_started：未开通。
- active：正常服务。
- suspended：已暂停。
- terminated：已终止。
- failed：上次操作失败，需要重试或人工处理。

执行操作前先点击预览，查看可操作数量、阻断原因和账务影响。终止、批量操作等危险动作需要二次确认。

提交 SIM 操作后，系统会生成 Operation ID。用户可在 SIM 详情中查看最近操作状态；如果操作失败，应根据失败原因处理账户状态、套餐授权或供应商异常后再重试。

## 6. eSIM 管理

eSIM 页面用于管理 Profile 和 eID/eUICC。可查看 Profile 状态，按 eID 分配 Profile，发起下载、启用、禁用、删除、释放等操作。生产环境中，eSIM 操作通常异步完成，最终结果以轮询、站内通知或 Webhook 返回。

当前 MVP 先支持启用和释放示例 Profile，用于验证前后端操作闭环。真实下载、SM-DP+ 地址、确认码和 eIM 编排后续接入供应商 Adapter。

## 7. 套餐与流量池

套餐支持地区、流量大小、计费起点、共享流量池、超额策略和价格版本。套餐状态包括 draft、pending_review、active、deprecated、retired、archived。

流量池用于多个 SIM 或账户共享流量，超额策略可以配置为限速、停机、按量计费或自动加购。

## 8. CDR 与用量

客户可以在 CDR 页面查看统一格式话单。平台会从资源方获取原始 CDR，标准化后提供给客户查询和导出。导入时重点校验 ICCID、用量字节数、供应商记录号、国家、运营商和开始时间。

当前页面提供“导入示例 CDR”按钮，便于联调验证导入、去重、标准化和列表刷新。正式文件上传和行级错误下载后续补充。

## 9. Billing

账单流程：

1. 系统按账期生成预览账单。
2. 财务检查套餐费、超额费、调整项和税费。
3. 审批通过后发布正式账单。
4. 系统生成 PDF 和明细文件。
5. 邮件发送给客户账单联系人。

已发布账单不应直接修改，错误通过调整单、贷项或借项处理。

当前页面提供账单预览和发送首张账单的最小操作，用于验证 Billing API、通知入队和状态刷新。

## 10. 批量操作

批量任务适用于 SIM 导入、批量暂停、批量套餐变更、批量 eSIM 操作、CDR 重跑等场景。建议流程：下载模板、填写上传、结构校验、业务校验、风险预览、审批、执行、下载行级结果。

当前页面支持创建示例批量任务并执行可执行任务。Excel 上传、模板下载和复杂审批暂缓，先保证任务创建、行级结果和状态刷新闭环。

## 11. API 与 Webhook

客户开发者可以在 API 与 Webhook 页面创建 API Client 和 Webhook 订阅。

API Client 使用建议：

1. 为生产、测试环境分别创建不同 Client。
2. 只勾选需要的 Scope。
3. 创建后立即保存一次性密钥，页面不会再次展示。
4. 定期轮换密钥。
5. 当系统停用或泄露风险出现时立即暂停 Client。

Webhook 使用建议：

1. 选择需要订阅的事件，例如 `sim.activated`、`sim.suspended`、`invoice.issued`。
2. 填写 HTTPS 回调地址。
3. 保存一次性签名密钥。
4. 使用测试推送验证接收服务。
5. 在接收端校验签名、时间戳和事件 ID，避免伪造和重放。
