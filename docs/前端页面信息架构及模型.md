# CMP 前端页面信息架构及模型

## 1. 设计基调

CMP 是高频运营型 SaaS 后台，不适合营销式页面。前端应优先支持快速查询、批量操作、状态判断、异常定位和财务核对。

界面原则：

- 信息密度中高，但层级清楚。
- 列表页支持高级筛选、保存视图、列配置、批量选择和导出。
- 详情页采用概览、配置、操作记录、用量、账单、审计的标签页结构。
- 高风险操作必须给出影响预览、二次确认和操作结果。
- 所有异步任务都显示状态、进度、失败原因和 correlation_id。

## 2. 顶层导航

建议一级导航：

- Dashboard
- SIM 与设备
- eSIM Profiles
- 套餐与流量池
- 用量与 CDR
- 账户与客户
- Billing 账单
- 批量任务
- 资源方
- API 与 Webhook
- Settings
- Audit Logs

不同角色看到的菜单不同。企业客户不应看到供应商成本、平台全局配置和其他租户数据。Reseller 可看到下级账户和自己的利润数据，但不能看到平台全局成本策略。

## 3. Dashboard 页面模型

### 平台 Dashboard

核心组件：

- KPI 卡片：活跃 SIM、活跃 eSIM、当月用量、收入、成本、毛利、失败操作。
- 用量趋势图：按日/小时展示 data usage。
- 地区分布：国家/区域用量排行。
- 供应商健康：API 成功率、平均延迟、失败数、CDR 延迟。
- 运营队列：批量任务、CDR 解析、Rating、账单运行。
- 告警列表：信用额度、异常漫游、用量突增、供应商异常。

### 客户 Dashboard

核心组件：

- 当前套餐和剩余流量。
- 流量池使用率。
- Top SIM 用量排行。
- 账单状态。
- eSIM 操作结果统计。
- 告警和通知。

## 4. SIM 与设备模块

### SIM 列表页

筛选：

- 账户、ICCID、IMSI、MSISDN、eID、IMEI、库存状态、服务状态、最近操作状态、供应商、国家、套餐、分组。

表格列：

- ICCID
- MSISDN
- IMSI
- 账户
- 库存状态
- 服务状态
- 最近操作状态
- 套餐
- 本月用量
- 国家/运营商
- 最后在线时间
- 供应商
- 操作

行操作：

- 查看详情
- 激活
- 暂停
- 恢复
- 变更套餐
- 加入分组
- 查看 CDR

批量操作：

- 激活、暂停、恢复、销户、变更套餐、导出、加入分组。

### SIM 详情页

标签：

- 概览：状态、账户、套餐、用量、网络信息。
- 用量：按小时/日/月趋势，区域分布。
- CDR：标准话单明细。
- 套餐：订阅记录、流量池、超额策略。
- 诊断：最后 attach、PDP session、IP、IMEI、漫游状态。
- 操作记录：生命周期和供应商操作。
- 审计：用户/API 操作记录。

详情页操作区需要根据状态机展示可执行操作。不可执行的操作不应只隐藏，建议在操作菜单中显示禁用原因，例如账户冻结、套餐不可用、供应商通道异常、存在未完成操作、需要审批。

## 6. 套餐与流量池模块

### 套餐列表页

字段：

- 套餐名称、区域、流量、周期、起算方式、流量池、套餐状态、当前价格版本、授权账户数、存量订阅数、创建时间。

操作：

- 新建套餐
- 发布新版本
- 复制套餐
- 下架
- 分配给 Reseller/客户

套餐列表需要区分 active、deprecated、retired。deprecated 不允许新客户订购但可保留存量；retired 不允许新开通和续期。

### 套餐配置向导

步骤：

1. 基础信息：名称、编码、类型、状态。
2. 覆盖区域：国家、Zone、运营商、MCC/MNC。
3. 流量规则：固定流量、按量、无限 FUP、阶梯。
4. 周期规则：自然月、开通日、首次使用、合同日。
5. 流量池：是否共享、共享范围、重置策略。
6. 超额策略：断网、限速、超额收费、自动加购。
7. 价格版本：币种、基础费、超额单价、Reseller 最低价。
8. 发布确认：影响客户、版本生效时间。

### 流量池详情页

内容：

- 池总量、已用、剩余、周期。
- 池内 SIM。
- 用量趋势。
- 超额预测。
- 告警策略。

筛选：

- 账户、时间、SIM 分组、国家、运营商、套餐、供应商。

图表：

- 流量趋势。
- 地区排行。
- SIM 排行。
- 套餐用量。
- 异常流量。

### CDR 列表页

字段：

- 时间、ICCID、IMSI、MSISDN、国家、运营商、上行、下行、总量、计费单位、金额、供应商、原始文件。

操作：

- 导出。
- 查看原始映射。

## 8. 账户与客户模块

### 账户树页面

左侧账户树，右侧账户详情。

功能：

- 创建下级账户。
- 设置账户类型。
- 分配套餐权限。
- 配置 Billing Profile。
- 配置 API 权限。
- 查看下级汇总用量和账单。

账户树节点需要同时展示业务状态、风控状态和财务状态。上级账户 suspended 或 credit_hold 时，下级节点应显示继承限制提示。

### 账户详情页

标签：

- 概览
- 下级账户
- 用户与角色
- 套餐权限
- SIM/eSIM
- Billing
- API Key
- 审计日志

账户详情页建议增加“状态与限制”标签，用于展示账户状态流转、暂停原因、信用额度、风控冻结、逾期情况、继承自上级的限制，以及恢复服务需要满足的条件。

## 14. 前端状态与权限模型

前端建议按领域组织状态：

- authStore：当前用户、账户范围、权限点。
- accountStore：账户树、当前账户。
- simStore：列表查询条件、选中项、批量状态。
- esimStore：Profile、操作状态。
- packageStore：套餐配置草稿。
- billingStore：账单运行、账单详情。
- batchStore：任务进度和逐行结果。

权限控制：

- 路由级控制。
- 菜单级控制。
- 按钮级控制。
- 数据范围控制由后端兜底，前端只做体验优化。

高风险操作前端统一走 preview -> approval/confirm -> submit -> track result 流程。激活、暂停、恢复、终止、批量套餐变更、账户关闭、套餐下架都应复用这个交互模型。
