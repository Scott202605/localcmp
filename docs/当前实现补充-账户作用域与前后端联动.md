# 当前实现补充：账户作用域与前后端联动

本文基于当前代码实现补充说明账户作用域、账户/用户创建、前端页面结构、后端权限边界、API、用户操作和部署验证要求。

## 1. 设计补充

当前实现将账户树作为 CMP 多租户隔离的核心边界。

- `platform` 是平台根账户，可在任意账户下创建下级账户。
- `reseller` 是代理账户，可在自身或下级 reseller 账户下创建 `reseller` 或 `customer`。
- `customer` 是最终客户账户，不能继续创建下级账户。
- 所有写操作都应遵守“当前账户及下级账户”范围，禁止给平行账户或上级账户创建数据。
- SIM、eSIM、套餐授权、流量池、API Client、Webhook、Billing、CDR 导出等模块后续都应复用同一账户树 Scope Guard。

账户创建资料已扩展为：

- 账户名称
- 账户编码
- 账户类型
- 父账户
- 公司名称
- 税号
- 联系人
- 联系人邮箱
- 公司地址
- 币种
- 时区

这些字段后续用于 Billing Profile、合同、发票、对账和合规审计。

## 2. API 文档补充

### 2.1 账户作用域请求头

前端切换顶部账号作用域后，会在 API 请求中携带：

```text
X-Account-Scope-Id: <account_id>
```

后端只接受 JWT 登录账户的自身或下级账户作为有效 scope。若请求写入平行账户或上级账户，接口返回：

```json
{
  "error": {
    "code": "ACCOUNT_SCOPE_DENIED",
    "message": "Target account is outside current account scope"
  }
}
```

### 2.2 账户 API

- `GET /api/v1/accounts`：获取账户列表。
- `GET /api/v1/accounts/tree`：获取账户树扁平结构。
- `GET /api/v1/accounts/{accountId}`：获取账户详情、下级账户、绑定用户、SIM 与套餐授权摘要。
- `POST /api/v1/accounts`：创建账户。
- `POST /api/v1/accounts/{accountId}/{action}`：账户状态流转。

创建账户请求示例：

```json
{
  "accountName": "FleetLink South",
  "accountCode": "FLEETLINK-SOUTH",
  "accountType": "customer",
  "parentAccountId": "acc_reseller_east",
  "companyName": "FleetLink Technology Ltd.",
  "taxId": "TAX-91310000",
  "contactName": "Jane Ops",
  "contactEmail": "jane@example.com",
  "companyAddress": "Shanghai, China",
  "currency": "CNY",
  "timezone": "Asia/Shanghai"
}
```

关键校验：

- `accountType` 只允许 `reseller` 或 `customer`。
- 父账户必须存在且为 `active`。
- 父账户必须是 `platform` 或 `reseller`。
- 父账户必须在当前请求账户作用域内。
- `accountCode` 必须唯一。

### 2.3 用户 API

- `GET /api/v1/users`：获取用户列表。
- `GET /api/v1/users/{userId}`：获取用户详情、账户授权与角色绑定。
- `POST /api/v1/users`：创建用户并绑定账户与角色。
- `POST /api/v1/users/{userId}/deactivate`：停用用户。

创建用户请求示例：

```json
{
  "displayName": "Jane Ops",
  "email": "jane@example.com",
  "phone": "13900000000",
  "accountId": "acc_fleetlink",
  "roleId": "role_viewer",
  "initialPassword": "LocalCMP@2026"
}
```

关键校验：

- 只能为当前账户或下级账户创建用户。
- 不允许为平行账户或上级账户创建用户授权。
- 邮箱必须全局唯一。
- 初始密码只在创建响应中返回一次。

### 2.4 其他已落地接口

当前阶段一还已落地：

- `POST /api/v1/packages`：创建套餐草稿。
- `GET /api/v1/packages/{packageId}`：获取套餐详情。
- `POST /api/v1/usage-pools`：创建流量池。
- `GET /api/v1/suppliers/{supplierId}`：获取资源方详情。
- `POST /api/v1/suppliers/{supplierId}/sync-products`：同步资源方产品状态。
- `POST /api/v1/settings`：保存安全、SLA、保留周期配置。
- `POST /api/v1/audit-logs/export`：创建审计导出任务。

## 3. 前端信息架构补充

### 3.1 顶部账号作用域

顶部账号切换已从静态下拉改为搜索自动补全：

- 支持按账户名称或账户编码搜索。
- 账号候选来自后端 `/accounts` 数据。
- 新账户创建成功后，会刷新候选列表，并自动进入新账户详情。
- Platform 账号可切换到任意账户。
- 非平台账号最终以后端账户树校验为准。

### 3.2 账户与客户模块

当前页只保留列表和入口按钮：

- 账户列表：展示当前作用域账户及下级账户。
- 用户列表：展示当前作用域账户树内用户。
- 创建账户按钮：进入独立创建页面。
- 创建用户按钮：进入独立创建页面。

点击账户行进入独立账户详情页面，详情页面展示：

- 基础资料
- 公司与联系人
- 父账户与账户路径
- 下级账户
- 绑定用户
- SIM/号码资源范围

### 3.3 创建页面

创建账户页面：

- 父账户选项只展示当前作用域内可创建下级的 `platform` 或 `reseller`。
- 当前作用域为 `customer` 时，不展示可提交的创建表单，并提示不可创建下级账户。

创建用户页面：

- 绑定账户选项只展示当前作用域及下级账户。
- 角色选项来自后端 `/roles`。

## 5. 用户手册补充

日常操作建议：

1. 登录系统。
2. 在顶部 Account scope 搜索框选择当前要管理的账户。
3. 进入“账户与客户”查看当前账户及下级账户。
4. 点击账户行进入详情页。
5. 若当前账户是 platform 或 reseller，可点击“创建账户”创建下级账户。
6. 若当前账户是 customer，则不能创建下级账户。
7. 点击“创建用户”可为当前账户或下级账户创建用户。
8. 如出现 `ACCOUNT_SCOPE_DENIED`，说明目标账户不在当前账号作用域内。

## 6. 运维与部署补充

前后端分离部署时，需要允许以下请求头：

```text
Authorization
X-Correlation-Id
X-Account-Scope-Id
Idempotency-Key
```

上线验证清单：

- 使用平台账号登录。
- 顶部 Account scope 搜索自动补全可用。
- 切换到 reseller 后，只显示该 reseller 及下级账户。
- 创建新 customer 后，账号搜索候选实时包含新账户。
- 新账户创建成功后自动进入详情页。
- 切换到 customer 后创建下级账户入口不可用。
- API 尝试为平行账户创建用户应返回 `ACCOUNT_SCOPE_DENIED`。
- API 尝试在 customer 下创建子账户应返回 `PARENT_ACCOUNT_NOT_RESELLER`。
